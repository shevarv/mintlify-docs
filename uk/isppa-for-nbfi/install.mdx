«ISPPA for NBFI FinAP CheckLists» розроблено засобами Java та СУБД PostgreSQL, що є мультиплатформенними і їх робота не має залежати від операційної системи, де Модуль буде встановлено (Windows, Linux, Unix тощо).

Для роботи програми необхідно:

1. Інсталювати JRE (Java SE Runtime Environment) на кожен комп’ютер, де буде виконуватися програма.
2. Інсталювати PostgreSQL (на локальний комп’ютер, або сервер).
3. Створити (відновити, restore) базу даних “nbfitest” з необхідними довідниками та переліками даних.
4. Розпакувати архів із програмою «CheckLists»

### Встановлення JRE (Java SE Runtime Environment 8).

JRE відповідно вашої операційної системи можна завантажити за [посиланням](https://www.oracle.com/ua/java/technologies/javase/javase8-archive-downloads.html).

### Установка клієнтського місця.

Програму можна скачати за [посиланням](https://finap.com.ua/ispafornbfi). Якщо робоче місце на базі ОС Windows, інсталювати програму можна за допомогою програми-інсталятора, у разі використання іншої ОС - розпакувати архів в робочу директорію програми. Для розпакування архіву з програмою скористайтеся стандартними архіватором 7z чи будь-яким іншим, що підтримує даний формат архіву (Мал. 2.3):

![2 3](/2.3.png)

**Мал.2.3**

Зазначте шлях, де буде знаходитися Модуль (Мал. 2.4):

![2 4](/2.4.png)

**Мал.2.4**

Після успішного розпакування, каталог, де розміщена програма буде мати вигляд (Мал. 2.5):

![2 5](/2.5.png)

**Мал.2.5**

де:

- in - каталог для розміщення файлів, призначених для імпорту (listofclient.csv)
- lib - каталог бібліотек Java out - каталог для розміщення файлів, що експортуються (наприклад, вихідних форм у форматі MS Excel, побудови графів, візуалізації грошових потоків тощо)
- template - каталог для розміщення шаблонів вихідних друкованих форм (в Модулі не використовується)
- doc - документація по роботі програми
- log - файли логування роботи програми
- updates – каталог для збереження файлів поновлення програми
- config.xml – файл конфігурації
- nbfi.jar – головний модуль «ISPPA for NBFI FinAP CheckLists».

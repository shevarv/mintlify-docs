#### Розгортання пулу серверів для доступу до сервісу CheckLists

<Badge color="blue" size="xs">2025-01-01</Badge>

<Info>
  **Для забезпечення безперебійного доступу до сервісу, в пулі наших серверів було додатково розгорнуто точки доступу.**\
  \
  Щоб мати доступ до сервісу CheckLists як по API так і через WEB ресурс, необхідно мати доступ по порту 9443 до даного пулу серверів:

  - [**res1.finap.com.ua**](http://res1.finap.com.ua) (194.247.12.98)
  - [**res2.finap.com.ua**](http://res2.finap.com.ua) (185.101.217.98)
  - [**res3.finap.com.ua**](http://res3.finap.com.ua) (176.241.128.101)

  Пріоритетним варіантом є надання доступу на основі DNS серверів до домену [finap.com.ua](http://finap.com.ua) і всіх його субдоменів (\*.[finap.com.ua](http://finap.com.ua)). Це дозволить в майбутньому не відкривати окремі доступи до нових серверів які будуть розгортатися в нашому пулі.
</Info>

#### Зміни структури в реєстрах

<Badge color="blue" size="xs" stroke>2024-12-01</Badge>

<Info>
  **З 01.01.2025 року (продовжено до 28.02.2025) в API CheckLists відбудуться наступні зміни:**\
  \
  **• Реєстр «Санкції»**   \
  \
  З результату запиту (result) будуть вилучені блоки sanctions та sanctions\_lost на користь нового блоку санкцій анкетного типу psanctions, який вже присутній в сервісі. Параметри listdata залишаються незмінними для запитів в даний реєстр.\
  \
  **• Реєстр «Реєстр юридичних осіб та ФОП»**   \
  \
  З результату запиту (result) буде вилучено блок edrinfo (21 біт параметру listdata). Для аналізу даних з реєстру Реєстр юридичних осіб та ФОП надалі можна використовувати блок edrfullinfo (22 біт параметру listdata), який містить більше інформації по ЮО чи ФОП.\
  \
  **Тим клієнтам, які при інтеграції API використовують дані блоки, необхідно зробити відповідні зміни до зазначеної дати !**\
</Info>
